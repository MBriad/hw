### 自适应彩蛋培养系统

彩蛋不使用预设固定关键词，也不在用户第一次输入某个词时立即触发。系统先观察用户日常记录，在积累足够样本后由 AI 提炼高频语义、相似表达和推荐场景；模式成熟后，未来再次检测到相似内容才触发彩蛋。

培养流程：`OBSERVING -> CANDIDATE -> MATURE -> COOLDOWN`。

- [x] 定义 `LearnedPattern`、`PatternSuggestion` 和 `PatternMatchResult` 领域模型，覆盖主题、别名、出现次数、不同日期数、置信度、培养阶段、效果建议和冷却状态。
- [x] 保留现有本地词频统计用于 Patterns 页统计，不再让词频里程碑直接触发正式打卡彩蛋。
- [x] 正常打卡保存后写入本地待分析观察队列；相同日期覆盖最新内容，空内容忽略，样本不足时不触发彩蛋。
- [x] 明确本地候选和成熟门槛：候选至少 3 次、2 个不同日期、置信度 0.55；成熟至少 5 次、3 个不同日期、置信度 0.8。
- [ ] 阶段性批量调用 AI 分析新增记录，不让每次打卡同步等待网络请求。
- [ ] 要求 AI 返回结构化结果：高频语义主题、相似表达、推荐彩蛋场景、效果标签、声音标签和置信度。
- [x] 使用 `LearnedPatternRepository` 和 Preferences Adapter 持久化本地模式，后续培养和匹配不依赖每次在线请求。
- [x] 仅允许 `MATURE` 模式触发彩蛋；`OBSERVING` 和 `CANDIDATE` 不触发反馈。
- [ ] 对未来输入执行真正的相似语义匹配；当前离线阶段仅支持 AI 建议中的主题和别名包含匹配。
- [x] 为成熟模式设置 24 小时触发冷却，并复用现有反馈重入阻断，避免同一彩蛋高频叠加。
- [ ] 允许模式随长期使用增强、衰减或重新进入观察状态。
- [ ] 验证横幅进入、停留、退出、手动关闭、重复触发和非阻塞交互。
- [x] 在 Profile 调试面板增加独立 `PATTERN LEARNING LAB`，覆盖阶段、匹配、冷却、只读 `OBSERVATION STATUS` 和 `RESET SAMPLE`。
- [ ] 在 Patterns 页面显示观察中、候选、已成熟状态，但不提前暴露或播放尚未成熟的彩蛋。
